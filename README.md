# Blog Customizer

**Ссылка на проект:** [https://github.com/murxsxk1/blog-customizer](https://github.com/murxsxk1/blog-customizer)

## Описание

Blog Customizer - это React-приложение для настройки внешнего вида статьи в режиме реального времени. Пользователь может изменять различные параметры отображения текста и фона через интерактивную боковую панель.

## Возможности

- **Настройка шрифта**: выбор семейства шрифтов
- **Размер шрифта**: изменение размера текста через радио-кнопки
- **Цвет шрифта**: выбор цвета текста
- **Цвет фона**: настройка цвета фона статьи
- **Ширина контента**: регулировка ширины области контента
- **Сброс настроек**: возврат к параметрам по умолчанию
- **Применение изменений**: сохранение текущих настроек

## Технологии

- **React 18** с TypeScript
- **SCSS/CSS Modules** для стилизации
- **clsx** для условного применения классов
- **Webpack** для сборки проекта

## Основные компоненты

### App

Главный компонент приложения, который:

- Управляет состоянием настроек статьи
- Применяет CSS-переменные для динамического изменения стилей
- Рендерит форму настроек и статью

### ArticleParamsForm

Боковая панель с формой настроек:

- Управляет собственным состоянием открытия/закрытия
- Содержит все элементы управления настройками
- Обрабатывает клики вне области и нажатие Escape для закрытия
- Передает настройки в родительский компонент через колбэк

## Особенности реализации

### Управление состоянием

- Использует React hooks (useState, useEffect, useRef)
- Локальное состояние в ArticleParamsForm синхронизируется с глобальным состоянием через колбэки
- Дефолтные значения хранятся в константах

### Стилизация

- CSS-переменные для динамического изменения внешнего вида
- CSS Modules для инкапсуляции стилей
- SCSS для удобства написания стилей

### Доступность

- Обработка клавиатурных событий (Escape)
- Семантическая разметка (aside, form, article)
- Правильные aria-атрибуты в UI-компонентах

### Оптимизация

- Предотвращение всплытия событий
- Правильная очистка event listeners
- Использование useRef для DOM-операций

npm run start

# Сборка для продакшена

## Конфигурация

Настройки по умолчанию и доступные опции определены в `src/constants/articleProps.ts`:

- `fontFamilyOptions` - доступные семейства шрифтов
- `fontSizeOptions` - размеры шрифтов
- `fontColors` - цвета текста
- `backgroundColors` - цвета фона
- `contentWidthArr` - варианты ширины контента
- `defaultArticleState` - состояние по умолчанию

## Архитектурные решения

- **Разделение ответственности**: каждый компонент отвечает за свою область
- **Переиспользуемые UI-компоненты**: все элементы интерфейса вынесены в отдельные компоненты
- **Типизация**: использование TypeScript для предотвращения ошибок
- **Модульность**: четкая структура папок и файлов
